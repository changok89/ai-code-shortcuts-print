Codex 치트시트

자동완성보다 작업 단위와 검증 루프 중심으로 쓸 때 필요한 실행 패턴 압축판.

v2 · color / mono toggle

핵심 명령

  • codex exec "task"
  • codex exec --full-auto "task"
  • Ctrl+C 중단
  • git repo 안에서 실행

작업 단위

  • 입력 범위 명시
  • 목표 명시
  • 검증 기준 포함
  • 출력 형식 지정

검증 루프

  • pytest
  • npm test
  • pnpm lint
  • tsc --noEmit

잘 맞는 일

  • 작은 리팩터링
  • 테스트 실패 원인 찾기
  • diff 요약
  • PR 설명 초안

좋은 요청 예

  • auth.ts와 token.ts 범위
  • 401 원인 후보 3개 정리
  • 재현 조건 포함
  • 필요 시 수정 후보 제시

실패 시 규칙

  • 테스트 실패 시 중단
  • 변경 파일 요약
  • 검증 결과 정리
  • 남은 리스크 명시

피할 것

  • 모호한 제품 설계 전체 위임
  • 문맥만 길고 목표 없는 요청
  • 검증 없이 결과 신뢰
  • 너무 큰 작업 한 번에 넘기기

실전 팁

  • 작업기처럼 쓰고 자동완성으로만 보지 않기
  • 중간 확인 가능한 크기로 쪼개기
  • 검증 가능한 일부터 맡기기

한 줄 요약

  • Codex는 작업 단위와 검증 루프가 붙을 때 가장 강함
  • 프롬프트 길이보다 작업 설계가 중요
추천 용도: 버그 수정, 리팩터링, 요약, 검증 루프 작업 단위를 선명히 줄수록 결과 편차가 줄어듦
