Claude Code

  • 메인 허브로 두기 좋음
  • 계획 → 실행 → 리뷰 분리
  • 룰/스킬 파일이 품질 유지에 중요
  • 장기 문맥 운영에 강함

Codex

  • 작업 범위를 짧고 선명하게
  • repo / branch / worktree 격리
  • 테스트/린트 검증 루프 필수
  • 실패 시 요약 보고 구조 필요

Gemini Code

  • 입력을 파일/로그/제약/질문 순으로 정리
  • 출력 형식을 먼저 고정
  • 실행기보다 정리기 역할이 잘 맞음
  • 계획/요약/리뷰 초안에 강함

Qwen Code

  • 짧은 반복형 CLI 흐름에 유리
  • history / completion 활용도가 큼
  • 짧은 템플릿 프롬프트가 잘 맞음
  • 입력 루프 최적화가 핵심

공통 원칙

  • 모델보다 역할 배치가 중요
  • 검증 없는 자동화는 금방 흔들림
  • 반복 작업은 스킬로 고정
  • 사람 승인 단계를 남겨두기

추천 조합

  • Claude: 허브
  • Codex: 구현
  • Gemini: 요약/계획
  • Qwen: 빠른 반복

입력 설계

  • scope
  • goal
  • validation
  • output format

실패 설계

  • 언제 멈출지
  • 무엇을 보고할지
  • 어디서 재시작할지
  • 누가 승인할지
