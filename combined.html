Claude Code

Claude Code 단축표

작업 위임, 범위 지정, 문서 정리, 리팩터링 지시에 맞춘 한 장 요약표.

v1 · A4 출력용

핵심 단축

  • /help 핵심 명령 보기
  • /clear 현재 대화 문맥 정리
  • /status 모델/세션 상태 확인
  • Ctrl+C 실행 중단

자주 쓰는 명령 패턴

  • claude --print "task"
  • claude --permission-mode bypassPermissions --print "task"
  • 구현 전 파일 범위와 완료 조건을 먼저 명시

작업 흐름 팁

  • 작업 목표, 범위, 검증 기준을 3줄로 나눠 주기
  • 문서/설명 초안은 먼저 받고 최종 마감은 사람이 검토
  • 큰 작업은 기능 단위로 쪼개서 순차 실행

이럴 때 좋음

  • 파일 범위가 비교적 명확한 구현
  • 문서 초안, 리팩터링, 구조 정리
  • 작업 위임과 결과 요약이 같이 필요한 경우

이렇게 쓰기

  • 관련 파일 경로를 먼저 지정
  • 테스트/검증 명령을 함께 넘기기
  • 출력 형식을 지정해서 요약 받기

피할 것

  • "알아서 다 해줘" 같은 모호한 요청
  • 한 번에 너무 넓은 설계/구현 범위 넘기기
  • 검증 없이 바로 커밋/배포로 이어가기
Codex

Codex 단축표

자동완성보다 작업 단위 중심으로 쓸 때 필요한 검증 루프와 실행 패턴을 정리한 한 장 요약표.

v1 · A4 출력용

핵심 단축

  • codex exec "task" one-shot 실행
  • codex exec --full-auto "task"
  • Ctrl+C 중단
  • git repo 안에서 실행하기

자주 쓰는 명령 패턴

  • codex exec "Summarize this diff"
  • codex exec "Fix failing tests in auth module"
  • codex exec --full-auto "Refactor and run tests"

작업 단위 설계

  • 입력 범위, 목표, 검증 기준을 같이 주기
  • 작은 리팩터링/원인 조사/초안 생성에 특히 잘 맞음
  • 큰 작업은 단계별로 쪼개고 중간 확인 넣기

검증 루프

  • pytest, npm test
  • pnpm lint, tsc --noEmit
  • 실패 시 변경 파일 / 검증 결과 / 남은 리스크 요약

이렇게 쓰기

  • 파일 범위를 지정하기
  • 완료 조건을 명확히 하기
  • 검증 루프 없이 끝내지 않기

피할 것

  • 모호한 제품 설계 전체를 한 번에 맡기기
  • 검증 없이 결과를 신뢰하기
  • 문맥이 너무 긴 상태로 계속 이어가기
Gemini Code

Gemini Code 단축표

요약, 계획, 리뷰, 초안 생성처럼 구조화된 출력이 필요한 흐름에 맞춘 한 장 요약표.

v1 · A4 출력용

핵심 단축

  • 새 작업마다 목표를 한 문장으로 선언
  • 컨텍스트는 파일/로그/요구사항 순으로 정리
  • 출력 형식을 먼저 지정

자주 쓰는 요청 패턴

  • Summarize this code path
  • Draft a refactor plan
  • Review this diff and list risks

작업 흐름 팁

  • 먼저 요약, 다음에 실행 가능한 계획으로 변환
  • 초안/정리/비교표 생성에 강점을 활용
  • 코드 수정은 검증 가능한 작은 단위로 제한

이렇게 요청하기

  • 위험 요인 3개
  • 단계별 실행 계획
  • 검증 체크리스트
  • 최종 요약 5줄

이렇게 쓰기

  • 리뷰용/문서용/요약용 역할을 명확히 분리
  • 긴 입력은 항목 단위로 재구성해서 주기
  • 사람 승인 단계를 끝에 남기기

피할 것

  • 출력 형식 없이 긴 맥락만 던지기
  • 코드 실행 책임까지 한 번에 넘기기
  • 검증 없이 설명만 보고 신뢰하기
