Claude Code

Claude Code 치트시트

작업 위임, 문서 정리, 범위 있는 구현과 리팩터링에 맞춘 압축형 한 장 요약.

v2 · 통합 출력

핵심 키

  • Ctrl+C 실행 중단
  • Ctrl+L 화면 정리
  • Ctrl+R 히스토리 검색
  • Ctrl+G 프롬프트 편집기

슬래시 명령

  • /help 도움말
  • /clear 문맥 정리
  • /status 상태 보기
  • /model 모델 전환

자주 쓰는 실행

  • claude --print "task"
  • claude --permission-mode bypassPermissions --print "task"
  • 범위/목표/검증 조건 같이 적기

이럴 때 강함

  • 문서 초안 / 설명 정리
  • 범위 있는 구현 작업
  • 리팩터링 방향 잡기
  • 작업 위임 + 결과 요약

입력 템플릿

  • 범위: 어떤 파일/폴더
  • 목표: 무엇을 바꿀지
  • 검증: 무엇으로 확인할지
  • 출력: 어떤 형식으로 받을지

좋은 습관

  • 검증 명령 함께 넘기기
  • 큰 작업은 단계로 쪼개기
  • 최종 승인 단계 남기기
  • 요약 형식 지정하기

피할 것

  • "알아서 다 해줘" 같은 요청
  • 한 번에 너무 넓은 범위
  • 검증 없이 커밋/배포
  • 파일 범위 없이 추상 지시

작업 위임 패턴

  • 읽기 → 수정 → 검증 → 요약 순으로 지시
  • 문서/설명은 먼저 초안 받고 다듬기
  • 반복 작업은 출력 형식 고정

한 줄 요약

  • Claude Code는 넓은 맥락 이해와 정리, 작업 위임에 강함
  • 범위를 선명히 줄수록 품질이 좋아짐
Codex

Codex 치트시트

자동완성보다 작업 단위와 검증 루프 중심으로 쓸 때 필요한 실행 패턴 압축판.

v2 · 통합 출력

핵심 명령

  • codex exec "task"
  • codex exec --full-auto "task"
  • Ctrl+C 중단
  • git repo 안에서 실행

작업 단위

  • 입력 범위 명시
  • 목표 명시
  • 검증 기준 포함
  • 출력 형식 지정

검증 루프

  • pytest
  • npm test
  • pnpm lint
  • tsc --noEmit

잘 맞는 일

  • 작은 리팩터링
  • 테스트 실패 원인 찾기
  • diff 요약
  • PR 설명 초안

좋은 요청 예

  • auth.ts와 token.ts 범위
  • 401 원인 후보 3개 정리
  • 재현 조건 포함
  • 필요 시 수정 후보 제시

실패 시 규칙

  • 테스트 실패 시 중단
  • 변경 파일 요약
  • 검증 결과 정리
  • 남은 리스크 명시

피할 것

  • 모호한 제품 설계 전체 위임
  • 문맥만 길고 목표 없는 요청
  • 검증 없이 결과 신뢰
  • 너무 큰 작업 한 번에 넘기기

실전 팁

  • 작업기처럼 쓰고 자동완성으로만 보지 않기
  • 중간 확인 가능한 크기로 쪼개기
  • 검증 가능한 일부터 맡기기

한 줄 요약

  • Codex는 작업 단위와 검증 루프가 붙을 때 가장 강함
  • 프롬프트 길이보다 작업 설계가 중요
Gemini Code

Gemini Code 치트시트

요약, 계획, 리뷰, 초안 생성처럼 구조화된 출력이 필요한 흐름에 맞춘 압축형 한 장 요약.

v2 · 통합 출력

핵심 원칙

  • 목표를 한 문장으로 선언
  • 맥락은 항목 단위로 정리
  • 출력 형식을 먼저 고정
  • 길게 주기보다 구조화해서 주기

자주 쓰는 요청

  • Summarize this code path
  • Draft a refactor plan
  • Review this diff and list risks
  • Turn notes into checklist

강한 영역

  • 요약
  • 계획
  • 리뷰 초안
  • 구조화된 문서 출력

출력 형식 예

  • 위험 요인 3개
  • 단계별 실행 계획
  • 검증 체크리스트
  • 최종 요약 5줄

이렇게 쓰기

  • 리뷰용/문서용/요약용 역할 분리
  • 긴 입력은 파일/로그/질문 순으로 재구성
  • 사람 승인 단계를 끝에 남기기

피할 것

  • 출력 형식 없이 긴 맥락만 던지기
  • 실행 책임까지 한 번에 넘기기
  • 검증 없이 설명만 보고 신뢰하기
  • 목표 없이 브레인스토밍만 길게 하기

실전 팁

  • 요약 → 계획 → 체크리스트 순으로 요청
  • 비교표, 구조 정리, 초안 생성에 활용
  • 코드 변경은 작은 단위로 검증 가능하게

모바일에서 보기

  • 카드가 1열로 접혀도 읽기 쉽게 설계
  • 짧은 문장 / 짧은 불릿 위주로 유지
  • 스크린샷보다 텍스트 밀도 우선

한 줄 요약

  • Gemini Code는 구조화된 출력이 필요한 흐름에서 특히 강함
  • 무엇을 어떤 형식으로 받을지 먼저 정할수록 좋음
