Gemini Code 치트시트

요약, 계획, 리뷰, 초안 생성처럼 구조화된 출력이 필요한 흐름에 맞춘 압축형 한 장 요약.

v2 · color / mono toggle

핵심 원칙

  • 목표를 한 문장으로 선언
  • 맥락은 항목 단위로 정리
  • 출력 형식을 먼저 고정
  • 길게 주기보다 구조화해서 주기

자주 쓰는 요청

  • Summarize this code path
  • Draft a refactor plan
  • Review this diff and list risks
  • Turn notes into checklist

강한 영역

  • 요약
  • 계획
  • 리뷰 초안
  • 구조화된 문서 출력

출력 형식 예

  • 위험 요인 3개
  • 단계별 실행 계획
  • 검증 체크리스트
  • 최종 요약 5줄

이렇게 쓰기

  • 리뷰용/문서용/요약용 역할 분리
  • 긴 입력은 파일/로그/질문 순으로 재구성
  • 사람 승인 단계를 끝에 남기기

피할 것

  • 출력 형식 없이 긴 맥락만 던지기
  • 실행 책임까지 한 번에 넘기기
  • 검증 없이 설명만 보고 신뢰하기
  • 목표 없이 브레인스토밍만 길게 하기

실전 팁

  • 요약 → 계획 → 체크리스트 순으로 요청
  • 비교표, 구조 정리, 초안 생성에 활용
  • 코드 변경은 작은 단위로 검증 가능하게

모바일에서 보기

  • 카드가 1열로 접혀도 읽기 쉽게 설계
  • 짧은 문장 / 짧은 불릿 위주로 유지
  • 스크린샷보다 텍스트 밀도 우선

한 줄 요약

  • Gemini Code는 구조화된 출력이 필요한 흐름에서 특히 강함
  • 무엇을 어떤 형식으로 받을지 먼저 정할수록 좋음
추천 용도: 요약, 계획, 리뷰, 구조화된 초안 출력 형식을 선명하게 잡을수록 품질이 안정적
